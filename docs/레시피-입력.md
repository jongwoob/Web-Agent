# 레시피 입력

`recipes:input:*` 명령은 사용자가 직접 사이트를 조작하며 알려 주는 순서를 레시피로 저장한다. 입력 중인 정보는 `work/recipe-input/active.json`에만 보관하고, 종료 전까지 공개 저장소에 추가하지 않는다.

## 기본 순서

1. 메타데이터 없이 입력을 시작할 수 있다.

   ```powershell
   npm run recipes:input:start
   ```

2. 이름, 설명, 호출 문구, 단계, URL, 메모를 필요한 만큼 추가한다. `|`로 여러 호출 문구나 메모를 함께 기록할 수 있다.

   ```powershell
   npm run recipes:input:add -- --name "기도 찬양" --description "기도할 때 재생할 찬양 재생목록" --provider youtube --trigger "기도 찬양 틀어줘"
   npm run recipes:input:add -- --step "YouTube 홈을 먼저 연 뒤 모두 재생을 누른다." --url "https://youtube.com/playlist?list=<playlist-id>"
   npm run recipes:input:add -- --note "로그인은 실제로 필요할 때만 사용자가 직접 진행한다."
   ```

3. 현재 입력 상태를 확인하거나 저장한다.

   ```powershell
   npm run recipes:input:status
   npm run recipes:input:finish -- --id "prayer-praise"
   ```

종료 시 기본 저장 위치는 `recipes/local/<provider>/`이다. 이 폴더는 Git에서 제외되므로 개인 URL, 개인 작업 순서, 테스트 기록이 실수로 공개 저장소에 올라가지 않는다. 테스트에 적은 확인·미확인 항목, 상태 파일, 스크린샷 경로도 생성된 Markdown에 함께 정리된다.

## 자동 실행 연결

입력한 레시피에 등록된 workflow 명령이 있으면 `active` 상태로 저장한다. 예를 들어 YouTube 재생목록 URL은 `workflow:youtube-playlist-play` 명령을 자동으로 연결한다. 등록된 workflow를 모르는 레시피는 `draft`로 저장되며, 기록된 순서를 수동으로 이어서 사용할 수 있다.

명령을 직접 지정할 수도 있다.

```powershell
npm run recipes:input:add -- --command "npm run workflow:youtube-playlist-play -- --playlist-url https://www.youtube.com/playlist?list=<playlist-id> --browser chrome"
```

## 공개 저장소 등록

검토한 범용 레시피만 `--publish`로 `recipes/<provider>/` 아래에 저장한다. 이 명령은 Git add, commit, push를 자동으로 수행하지 않는다.

```powershell
npm run recipes:input:finish -- --publish --id "shared-recipe-id"
```

비밀번호, OTP, cookie, 연결 키, 개인 식별 정보, 결제 정보는 단계·메모·테스트 기록에 입력하지 않는다. 외부 전송, 제출, 공유, 구매, 삭제, 게시, 계정 변경, 민감 정보 입력은 레시피를 저장했더라도 실행 직전에 별도 승인이 필요하다.

## Codex 대화 규칙

Codex에서 `레시피 입력 시작`이라고 말하면 입력 상태를 시작한다. 입력 중에는 사용자가 알려 주는 이름, URL, 페이지에서 수행한 행동, 확인 결과를 순서대로 기록한다. 사용자가 `레시피 입력 종료`라고 말하면 기본적으로 로컬 `recipes/local/`에 저장하고 결과 위치를 알린다. 공개 레시피로 등록하려면 사용자가 명시적으로 공개 저장을 요청해야 한다.
