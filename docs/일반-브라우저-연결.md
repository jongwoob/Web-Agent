# 일반 Chrome과 Edge 연결

일반 사용자가 이미 사용 중인 Chrome 또는 Edge에서 Web-Agent 레시피를 실행하려면 `user-browser-bridge` 확장을 한 번 설치한다. 이 방식은 기존 프로필을 복제하거나 원격 디버깅 포트를 일반 브라우저에 열지 않는다.

## 최초 설정

1. 저장소 루트에서 아래 명령을 실행한다. 일반 Chrome의 확장 관리 페이지가 열린다.

   ```powershell
   npm run workflow:user-browser-bridge-setup -- --browser chrome
   ```

2. 확장 관리 페이지에서 개발자 모드를 켠 뒤 `압축해제된 확장 프로그램을 로드합니다`를 선택한다.
3. `extensions/user-browser-bridge` 폴더를 선택한다.
4. 확장 아이콘을 열고 `work/user-browser-bridge/chrome-setup.json`에 저장된 `endpoint`와 `pairingKey`를 입력해 저장한다.
5. YouTube 레시피를 사용하려면 확장 팝업의 `YouTube 권한`을 눌러 해당 사이트 권한을 허용한다. 다른 사이트는 그 사이트를 연 상태에서 `현재 사이트 권한`으로 개별 허용한다.

Edge에서는 `--browser edge`로 설정 명령을 실행하고 같은 확장 폴더를 Edge에 로드한다. 설정 파일과 연결 키는 브라우저별로 `work/user-browser-bridge/`에 분리 저장되며 Git에 포함되지 않는다.

## 사용 방식

일반 브라우저 연결을 지원하는 레시피는 기본적으로 일반 Chrome을 사용한다. YouTube 재생목록은 아래처럼 실행한다.

```powershell
npm run workflow:youtube-playlist-play -- --playlist-url "https://www.youtube.com/playlist?list=<playlist-id>" --browser chrome
```

작업은 YouTube 홈을 먼저 방문한 뒤 재생목록으로 이동한다. 같은 재생목록 탭이나 빈 탭이 있으면 재사용하고, 재생 중인 동영상 탭과 다른 사이트 탭은 교체하지 않는다. 재사용 가능한 탭이 없을 때만 이미 열린 일반 Chrome 또는 Edge에 새 탭을 추가한다.

기존 전용 제어 세션이 필요한 경우에만 `--session controlled`를 명시한다.

## 범위와 안전성

- 확장은 기본 설치 시 인터넷 사이트 접근 권한을 받지 않는다. YouTube 또는 사용자가 현재 사이트에서 직접 허용한 origin만 접근할 수 있다.
- Web-Agent는 `127.0.0.1`에만 바인딩한 연결과 브라우저별 연결 키를 함께 확인한다. 처음 연결한 확장 ID를 고정해 다른 확장의 연결을 거절한다.
- 확장은 외부에서 내려받은 코드나 모델이 만든 임의 JavaScript를 실행하지 않는다. 저장소에 포함된 레시피별 작업만 처리한다.
- 비밀번호, OTP, cookie, 복구 코드, 기존 브라우저 프로필은 읽거나 복사하지 않는다. 로그인과 다중 인증은 사용자가 일반 브라우저 창에서 직접 처리한다.
- 전송, 제출, 공유, 구매, 삭제, 계정 변경, 게시, 민감 정보 입력은 일반 브라우저 연결을 사용하더라도 최종 화면을 준비한 뒤 명시적인 승인을 받아야 한다.

Chrome Manifest V3의 `scripting` API는 확장 권한을 가진 탭에서만 함수를 실행할 수 있고, 사이트 권한은 `optional_host_permissions`와 사용자 동의로 개별 요청한다. 관련 구현은 [Chrome Extensions 문서](https://developer.chrome.com/docs/extensions/reference/api/scripting)와 [권한 API 문서](https://developer.chrome.com/docs/extensions/reference/api/permissions)를 따른다.

## 확인 범위

YouTube 레시피는 실제 동영상의 재생 상태, 음소거, 탭 음소거, 볼륨, 재생 시간 진행까지 확인한다. 운영체제 스피커, 선택된 출력 장치, Windows 앱 볼륨은 브라우저 페이지에서 확인할 수 없다.
